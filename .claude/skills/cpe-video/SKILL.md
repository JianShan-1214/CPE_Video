# CPE New — 從零建立教學影片

從使用者提供的 C++ 程式碼與題目說明，自動切片、生成設定檔、逐步確認、產生音檔。

---

## Phase 1：環境檢查

執行以下三項檢查，**任一失敗就停下來**，告訴使用者怎麼修，等他修好再繼續：

1. `public/` 目錄存在（確認在專案根目錄執行）
2. `.env` 檔案存在
3. `.env` 中 `GOOGLE_APPLICATION_CREDENTIALS` 有設定，且該路徑的檔案存在

若全部通過：
```
✓ 環境檢查通過，開始建立影片。
```

---

## Phase 2：收集素材

依序詢問使用者（逐一問，不要一次全問）：

**問 1：資料夾名稱**
```
請輸入影片的資料夾名稱（英文、底線，例如 bubble_sort）：
```
確認 `public/<folder>/` 不存在（若已存在，警告使用者並詢問是否覆蓋）。

**問 2：題目說明**
```
請貼上題目說明或程式描述（任何格式皆可，可以是 OJ 題目、一段文字說明、或自己寫的描述）：
```

**問 3：完整 C++ 原始碼**
```
請貼上最終完整的 C++ 原始碼：
```

## Phase 3：AI 分析與切片

拿到素材後，進行以下分析：

### 3.1 理解程式結構

閱讀程式碼，辨識出教學上有意義的「里程碑」，例如：
- 引入標頭、宣告命名空間
- 定義輔助函式（每個函式一步或分幾步）
- main 函式的各個邏輯段落
- 輸入、處理、輸出分開說明

**切片原則：**
- 每步新增 1–5 行，確保每步都有清楚的教學重點
- 第一步（step 01）固定只放題目說明注解，程式碼為空或只有框架
- 最後一步為完整程式碼
- 合理步數：通常 6–15 步，依程式長度調整

### 3.2 規劃每步內容

對每個步驟決定：

| 欄位 | 說明 |
|------|------|
| `file` | `codeNN.cpp` |
| `label` | 簡短標題（在時間軸顯示） |
| `from` / `to` | 時長（秒）：基礎 5 秒，每多 1 行加 0.8 秒，上限 12 秒 |
| `highlight.startLine` / `endLine` | 這步新增的行範圍（1-indexed） |
| `highlight.color` | 依性質選色（見下方色表） |
| `subtitle` | 引導式旁白（見下方規則） |

**highlight 色表：**
- `blue`：一般說明、宣告
- `yellow`：迴圈、流程控制
- `red`：條件判斷
- `green`：輸出、關鍵操作
- `lightblue`：函式宣告

**subtitle 規則（引導式）：**
- 以「接下來」「這裡」「我們」等引導詞開頭
- 說明「做什麼」與「為什麼」
- 長度：30–60 字，適合 5–10 秒語音
- 第一步（題目說明）：將使用者提供的說明改寫成引導式介紹，例如「這題給我們 … 我們的目標是 … 接下來看看怎麼實作。」

## Phase 4：逐步確認

**每次只展示一步**，等使用者確認後再顯示下一步。

格式：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Step N / M  —  <label>
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
新增行：<startLine>–<endLine>
時長：<from>–<to> 秒
Highlight：<color>

程式碼片段（新增的部分）：
<顯示這步新增的程式碼>

Subtitle：
「<subtitle 全文>」
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[Enter] 確認  /  輸入修改要求
```

使用者回應處理：
- 按 Enter 或說「OK」→ 繼續下一步
- 說「改 subtitle」或直接給新文字 → 更新後重新顯示
- 說「調 highlight」→ 詢問新的行範圍或顏色
- 說「重新切」→ 回到 Phase 3 重新分析（可附帶說明期望的切法）

所有步驟確認完後：

```
✓ 全部 <M> 步確認完成。

即將產生：
  public/<folder>/code01.cpp … code<M>.cpp
  public/<folder>/config.json

確認產生檔案？[Y/n]
```

---

## Phase 5：產生檔案

建立 `public/<folder>/` 目錄，依序寫入：

1. `code01.cpp` … `codeNN.cpp`
   - 每個檔案是**累加的**：codeN 包含前 N 步所有程式碼
   - 保留縮排與原始格式

2. `config.json`，格式：
```json
{
  "steps": [
    {
      "label": "...",
      "from": 0,
      "to": 6,
      "file": "code01.cpp",
      "subtitle": "...",
      "highlight": {
        "startLine": 1,
        "endLine": 3,
        "color": "blue"
      }
    }
  ]
}
```

完成後顯示：

```
✓ 檔案產生完成！

  public/<folder>/
    config.json
    code01.cpp … code<M>.cpp

建議先預覽：
  npm run dev

確認畫面沒問題後，說「生成音檔」我就開始產生語音。
```

---

## Phase 6：生成音檔

等使用者說「生成音檔」「OK 生成」或類似確認語，再執行：

```bash
npm run gen-audio <folder>
```

顯示即時輸出。完成後：

```
✓ 音檔產生完成！

下一步：
  npm run render <folder>   # 輸出最終影片
```

---

## 守則

- **不跳過環境檢查** — Phase 1 失敗就停，不繼續
- **逐步確認** — Phase 4 每次只顯示一步，不一次全列
- **累加式 cpp 檔** — 每個 cpp 檔包含所有前面步驟的程式碼，不只是 diff
- **不主動 render** — render 讓使用者自己決定時機
- **gen-audio 等確認** — 預覽後使用者說要才跑，不自動觸發
